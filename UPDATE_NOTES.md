# LightNovelSelector v1.3.0 更新说明

发布日期：2026-06-18

## 更新重点

v1.3.0 是一次围绕“安全分类”和“高级感 UI”的稳定性更新。它整合了最近几次改进：深色现代工作台、非线性动画反馈、完整文件哈希重复检测、可撤销报告增强、设置保存容错、Windows 启动路径修复，以及完整中文 README。

## UI 与交互

- 主界面升级为深色石墨风格，减少传统 Tkinter 的粗糙感。
- 增加顶部运行状态胶囊，显示待扫描、扫描中、预览完成、分类完成等状态。
- 增加底部下一步操作提示，帮助用户明确当前应该扫描、修正、执行还是撤销。
- 统计卡片显示文件总数、可执行、重复、错误，并使用 `easeOutCubic` 数字缓动。
- 初始界面加入 `easeOutBack` 小幅进入动画。
- Toast 使用弹性滑入、渐入渐出和左侧状态色条。
- 进度条使用自绘胶囊样式，扫描和详情预加载阶段都有非线性动态反馈。
- 详情面板新增当前文件、状态、目标、来源，让分类依据更清楚。
- 表格行支持双击修正分类。
- 新增快捷键：
  - `Ctrl+O`：选择目录
  - `F5`：扫描并预览
  - `Ctrl+Enter`：执行分类
  - `Ctrl+R`：打开最近报告
  - `Ctrl+Z`：撤销上次分类

## 分类与文件安全

- 重复检测改为完整文件 SHA-256 哈希，不再只依赖文件头尾片段，降低误判不同文件为重复的风险。
- 重复项默认标记为 `duplicate` 并跳过，避免误移动重复副本。
- 如果文件移动到一半失败，会在重新抛出错误前写出部分 `classification_report.json`。
- 已经移动成功的文件会记录实际目标路径，可以通过报告撤销。
- 分类报告继续记录移动、跳过、重复、错误、置信度和实际目标路径。
- 单文件异常会进入错误状态，尽量不影响整批扫描。

## 设置与规则

- 新增 `try_save_app_settings()`，设置保存失败时不阻断扫描主流程。
- 设置页保存失败也会提示并写入日志，不会让界面线程直接异常退出。
- 自定义规则继续支持：

```text
匹配模式 => 系列名
```

示例：

```text
*SAO* => Sword Art Online
*无职转生* => 无职转生
```

## Windows 环境修复

- 修复 `run.bat` fallback Python 路径未加引号的问题。
- 用户名或 Python 路径包含空格时，启动命令也能正确执行。
- 保留 `build_exe.bat` 对损坏 `.venv-build` 的自动检测和重建能力。
